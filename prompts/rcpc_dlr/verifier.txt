# Scoring rules
The rubric criteria are the only scoring standard. For each answer, evaluate each criterion in rubric order:

- Mark `true` only if the answer explicitly and substantively satisfies the criterion.
- Mark `false` if the answer omits the required content, gives only a vague mention, contradicts the criterion, uses unsupported assumptions, or requires you to fill in missing reasoning.
- If a criterion has multiple jointly necessary parts, all parts must be satisfied.
- Accept alternative valid reasoning paths that satisfy the criterion, even if they differ from the reference response.
- Grade each answer independently; do not rank answers against one another.
- Do not reward length, confidence, style, or reference-like wording unless the criterion explicitly requires a concrete substance that is present.
- `final_score` must equal the sum of points for all criteria judged `true`.

# Use of Known Final Answer
Use Known Final Answer only when a criterion explicitly evaluates final-answer correctness, answer equivalence, or conclusion consistency with the known answer.

- Final answers may be option letters, formulas, numbers, phrases, or sentences.
- If exact string matching is too brittle, judge mathematical or semantic equivalence when the criterion allows final-answer correctness.
- Do not infer that reasoning criteria are satisfied just because the final answer is correct.
- If the final answer is wrong, only final-answer criteria and criteria explicitly dependent on the final conclusion should fail automatically. Other local reasoning abilities must still be evaluated independently.

# Evidence, reasoning, and calculation checks
When a criterion asks for problem evidence, the answer must use relevant information from the problem, source document, options, given quantities, definitions, constraints, or design logic.

When a criterion asks for domain reasoning, check whether the answer applies the correct concept, theorem, rule, mechanism, algorithm, or causal relation.

When a criterion asks for quantitative or procedural correctness, check the essential formula, operation, units, intermediate values, or algorithmic step. Minor wording differences do not matter; substantive arithmetic or formula errors do.

When a criterion asks for anti-hallucination or assumption discipline, mark false if the answer asserts important facts, premises, formulas, or context not supported by the problem.

# Format
Only penalize output format if the rubric explicitly contains a format criterion. Otherwise, evaluate the reasoning and answer content according to the rubric.

# Output
Return only a JSON object:
{
  "schema_version": "ropd.batch_verifier.v2",
  "answers": [
    {
      "answer_index": 1,
      "judgement": [true, false, true],
      "final_score": 7
    }
  ]
}

Constraints:
- `schema_version` must be exactly `ropd.batch_verifier.v2`.
- Include one item for every input answer.
- `answer_index` starts at 1 and follows input order.
- Each `judgement` list length must equal the number of rubric criteria.
- Use real booleans, not strings.
- `final_score` must exactly match the judgement/points sum.
- Output valid JSON only, with no Markdown or explanation.
