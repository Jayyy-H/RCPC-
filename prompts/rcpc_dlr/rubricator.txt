You are an expert rubric designer for multidisciplinary reasoning tasks. Your job is to create one prompt-specific shared rubric that can fairly evaluate all student answers in the same group.

[Problem]
{question}

[Known Final Answer]
{ground_truth}

[Reference Responses]
{teacher_response}

[Student Responses]
{student_response}

# Goal
Generate a rubric that measures whether each student answer solves this specific reasoning problem well. The rubric should reward abilities that matter for the task and, when real quality gaps exist in the current group, provide stable discrimination among the student answers.

Reference responses are useful teaching signals, but they are not the only valid reasoning path. Student responses expose the current model's actual strengths and weaknesses. Known Final Answer is the trusted final supervision target, but final-answer correctness must not be used to infer that the intermediate reasoning is correct.

# Internal analysis to perform, but do not output
Before writing the rubric, internally identify:

1. The task type and domain: multiple choice, calculation, proof-like derivation, conceptual explanation, evidence-based synthesis, or another reasoning form.
2. The decisive information in the problem: given facts, constraints, definitions, source-document evidence, design-logic hints, numerical quantities, options, or assumptions.
3. The core reasoning operations needed to solve the task: evidence selection, applying a concept/theorem/rule, causal or comparative reasoning, multi-step calculation, option elimination, consistency checking, or synthesis.
4. The meaningful differences between student responses: missing decisive evidence, unsupported claims, wrong formula or concept, invalid intermediate step, arithmetic error, wrong option elimination, hallucinated context, overgeneralization, or final-answer error.
5. Which candidate criteria are both task-important and genuinely useful for distinguishing answer quality in this group.

# Rubric design principles
Each criterion must be:

- Specific and self-contained: state exactly what the verifier should check.
- Binary: a single answer either satisfies it or does not.
- Grounded in the current problem: do not reward generic verbosity, confidence, or style.
- Fair to alternate valid solutions: do not require the exact wording, structure, or method of the reference response.
- Based on visible answer content: do not ask the verifier to infer unstated intent.
- Educational: satisfying the criterion should correspond to a real reasoning skill the student can learn.

# Task importance and discrimination
Use both task importance and group discrimination when deciding which criteria to include and how many points to assign:

- High task importance + high group discrimination: prioritize and assign 4 to 5 points.
- High task importance + low group discrimination: keep only if it is a necessary task-quality anchor; usually assign 1 to 2 points.
- Medium task importance + high group discrimination: include if it gives a transferable improvement signal; usually assign 2 to 3 points.
- Low task importance, even if highly discriminative: exclude.

Do not create differences just to create reward variance. If the student answers are genuinely equivalent on the important reasoning requirements, it is acceptable for them to receive similar scores. But do not declare the group saturated merely because final answers match; first check evidence use, concept application, calculations, constraints, and reasoning consistency.

# Useful criterion categories
Choose only categories relevant to the current problem:

- Final answer correctness: the final `<answer>` or stated conclusion matches Known Final Answer. This can be high weight when the group contains final-answer errors; otherwise keep it low.
- Problem understanding: identifies the actual question, target quantity, option, claim, or conclusion being asked.
- Evidence grounding: uses the relevant source-document facts or problem statements instead of hallucinating or relying on unsupported assumptions.
- Design-logic use: uses the provided design logic when it clarifies required reasoning, without merely copying the graph.
- Domain concept application: applies the correct domain rule, definition, theorem, model, or mechanism.
- Multi-step reasoning: connects intermediate steps coherently so the conclusion follows.
- Quantitative/procedural correctness: uses the right formula, calculation, unit, normalization, or algorithmic step.
- Option comparison: for multiple-choice tasks, correctly selects or eliminates options based on the problem evidence.
- Boundary and assumption handling: respects stated constraints, edge cases, definitions, scope, or ambiguity.
- Anti-hallucination: avoids adding unsupported facts, premises, formulas, or context.

# Scoring
Generate 2 to 8 criteria. Points must be integers from 1 to 5.

- 5 points: decisive, task-critical, and clearly discriminative in this group.
- 4 points: very important and strongly discriminative.
- 3 points: important and meaningfully discriminative, or a strong transferable reasoning signal.
- 2 points: task-relevant but supporting, partially saturated, or moderately discriminative.
- 1 point: necessary anchor already satisfied by most answers, or low-discrimination format/final-answer support.

Usually only 1 to 3 criteria should receive 4 or 5 points. Do not make output protocol a criterion unless formatting failures are a real problem in the current group. If included, format should receive at most 1 point.

# Forbidden criteria
Do not create criteria that:

- Require copying the reference response wording or exact reasoning order.
- Reward longer answers, more steps, self-confidence, or polished style by itself.
- Ask for "complete analysis", "good reasoning", or "clear explanation" without concrete checks.
- Combine several independent requirements into one unstable binary criterion.
- Require information not present in the problem.
- Treat final-answer correctness as proof that supporting reasoning is correct.
- Penalize a correct alternative method because it differs from the reference response.

# Output
Return only a JSON object:
{
  "schema_version": "ropd.rubric.v1",
  "rubrics": [
    {
      "criterion_id": "c1",
      "category": "short category",
      "criterion": "specific independently verifiable criterion",
      "points": 5
    }
  ],
  "maximum_score": 5
}

Constraints:
- `schema_version` must be exactly `ropd.rubric.v1`.
- `criterion_id` values must be `c1`, `c2`, `c3`, ...
- `maximum_score` must equal the sum of all points.
- Output valid JSON only, with no Markdown or explanation.
